# 記事作成ワークフロー

あなたは技術記事作成のオーケストレーターです。以下のワークフローに従って記事を作成してください。

## 引数

`$ARGUMENTS` には記事のテーマが渡されます。例：「Twig 3.24の新機能」「EC-CUBEのセキュリティ対策」

---

## ワークフロー

### Step 1: 情報収集

テーマに関する最新情報を収集します。

- WebSearchで最新のニュースや公式ドキュメントを検索
- 必要に応じて公式サイトをWebFetchで取得

### Step 2: 下書き作成

以下のルールに従って記事を作成します。

**ファイル配置:**
- `articles/` ディレクトリに作成
- ファイル名: 半角英数字とハイフンのみ（12〜50文字）

**フロントマター:**
```yaml
---
title: "記事タイトル"
emoji: "適切な絵文字"
type: "tech"
topics: ["eccube", "eccube4", "php", "symfony"]
published: false
---
```

**冒頭の注意書き（必須）:**
```markdown
:::message
この記事は EC-CUBE 4.3 以上を対象としています。
また、[Claude Code](https://claude.ai/claude-code) を使って書かれています。内容に誤りがある場合はコメントでお知らせください。
:::
```

**末尾のメッセージ（必須）:**
```markdown
---

:::message alert
**EC-CUBEのカスタマイズをお待ちしております！**

EC-CUBEのカスタマイズや開発のご相談は、お気軽にお問い合わせください。

この記事が役に立ったら、ぜひ**バッジを贈っていただけると励みになります！**
:::
```

### Step 3: 並列レビュー

下書き完成後、3つのレビューを**並列で**実行します。

必ず以下の3つのTaskを**同時に**呼び出してください：

```
Task 1: 正確性チェック
- subagent_type: general-purpose
- prompt: |
    以下の記事の技術的な正確性をチェックしてください。
    - コード例が正しいか
    - APIの使い方が正しいか
    - バージョン情報が正確か
    - 参考リンクが有効か

    問題があれば具体的な修正案を提示してください。

    記事パス: {記事のパス}

Task 2: セキュリティチェック
- subagent_type: general-purpose
- prompt: |
    以下の記事のセキュリティ観点でのチェックをしてください。
    - 脆弱性を生むコード例がないか
    - セキュリティのベストプラクティスに従っているか
    - 機密情報が含まれていないか

    問題があれば具体的な修正案を提示してください。

    記事パス: {記事のパス}

Task 3: バズ改善提案
- subagent_type: general-purpose
- prompt: |
    以下の記事をよりバズりやすくする改善案を提案してください。

    チェックポイント:
    - タイトルにインパクトがあるか（数字、問いかけ、対立構造）
    - 冒頭で読者を引き込めているか
    - 結論が先に書かれているか（PREP法）
    - SNSでシェアされやすい構成か
    - 議論を誘発する問いかけがあるか

    具体的な改善案（タイトル案、構成の変更など）を提示してください。

    記事パス: {記事のパス}
```

### Step 4: フィードバック統合

3つのレビュー結果を統合して、記事を改善します。

- 正確性の問題 → 必ず修正
- セキュリティの問題 → 必ず修正
- バズ改善提案 → タイトルと冒頭は採用、その他は適宜判断

### Step 5: PR作成

改善した記事をコミットしてPRを作成します。

```bash
git checkout -b article/{スラッグ}
git add articles/{ファイル名}
git commit -m "Add article about {テーマ}"
git push -u origin article/{スラッグ}
gh pr create --title "{記事タイトル}" --body "..."
```

### Step 6: 完了報告

ユーザーに以下を報告します：

- PRのURL
- 記事の概要
- 3つのレビューで指摘された主な点と対応内容

---

## 注意事項

- `published: false` で作成する（公開はユーザーが判断）
- EC-CUBE 4.3以上を対象とする
- コード例は実際に動作するものを記載
- 公式ドキュメントの内容と矛盾しないようにする
