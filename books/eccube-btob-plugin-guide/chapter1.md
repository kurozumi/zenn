## BtoBとBtoCの違い

まずは「BtoB」と「BtoC」の違いを整理しておきましょう。

**BtoC（Business to Consumer）** は、企業が一般消費者に商品やサービスを販売する形態です。Amazon や楽天でショッピングするのが典型的な例です。購入者は個人で、その場でクレジットカードや電子マネーで決済し、自宅に届いた商品を受け取る、というシンプルなフローです。

**BtoB（Business to Business）** は、企業が別の企業（法人）に商品やサービスを販売する形態です。たとえば食材メーカーがレストランに食材を卸す、印刷会社が広告代理店に印刷物を納品する、といったケースです。

BtoC と BtoB では、取引の性質が大きく異なります。

| 比較項目 | BtoC | BtoB |
|---|---|---|
| 購入者 | 個人 | 企業・法人 |
| 価格 | 定価・公開価格 | 取引先ごとの卸価格・特別価格 |
| 決済方法 | クレジットカード・電子マネーなど | 銀行振込・後払い（掛け払い）が中心 |
| 審査 | 基本なし（誰でも購入可） | 与信審査・取引先審査あり |
| 注文単位 | 1個から | ロット単位・箱単位が多い |
| 請求書 | 基本不要 | 請求書・納品書の発行が必須 |

特に重要な概念として「掛け払い（かけばらい）」があります。これは「今月購入した分を翌月末にまとめて支払う」という後払い方式で、BtoBでは非常に一般的です。消費者が即時決済するBtoCとは根本的に異なる取引スタイルです。

また「卸価格（おろしかかく）」とは、小売業者や取引先企業向けに設定された仕入れ値です。一般消費者には定価で販売する商品でも、法人取引先には定価の7掛け（30%オフ）などの特別価格で提供するケースが多くあります。

## EC-CUBE の標準機能でできること

EC-CUBEは日本のEC業界で広く使われているオープンソースプラットフォームです。標準機能として以下のような機能が揃っています。

**商品管理**
商品の登録・編集・削除、カテゴリ管理、在庫管理、商品規格（サイズ・カラーなど）の設定が可能です。複数の商品画像を登録でき、販売期間の設定もできます。

**注文管理**
受注一覧の確認、注文ステータスの管理（入金待ち・発送準備中・発送済みなど）、注文内容の編集、キャンセル処理などが行えます。

**会員管理**
会員の登録情報の確認・編集、会員へのメール送信、退会処理などができます。

**支払い方法・配送方法の設定**
クレジットカード、銀行振込、代金引換などの支払い方法と、宅急便、ゆうパックなどの配送方法を管理画面から設定できます。

**クーポン・セール機能**
割引クーポンの発行、期間限定セール価格の設定、まとめ買い割引なども標準で対応しています。

これらの機能を使えば、一般的なBtoCのECサイトであれば十分に構築できます。

## EC-CUBE 標準機能で BtoB に対応できない 5 つの壁

しかし、BtoBサイトを構築しようとすると、標準機能だけでは越えられない壁が5つあります。

### 壁1: 会員登録が誰でも即時に完了してしまう問題

EC-CUBEの標準機能では、会員登録フォームに必要事項を入力してメール認証を済ませると、それだけで会員登録が完了します。

BtoBサイトでは「取引先として適切な企業かどうか」を事前に確認したいはずです。たとえば「飲食店経営者のみに食材を卸したい」という場合、飲食店以外の一般消費者が登録できてしまっては困ります。また、反社会的勢力や信用状態の悪い企業との取引を避けるためにも、登録申請を受け付けてから管理者が審査し、承認した場合のみ利用を許可する仕組みが必要です。

標準機能にはこの「審査して承認する」フローが存在しません。

### 壁2: 全員に同じ価格が表示される問題

EC-CUBEの標準機能では、同じ商品は誰が見ても同じ価格が表示されます。

BtoBサイトでは取引先ごとに価格が異なります。「大手取引先のA社には特別価格で提供したい」「B社は通常の卸価格、C社はさらに優遇した価格」など、取引先の規模や関係性によって価格が変わるのが一般的です。

標準機能では価格の出し分けは一切できません。柔軟な卸価格の設定や、グループごとに異なる価格を商品ごとに細かく設定するには別途プラグインが必要です。

### 壁3: 全員に同じ商品が見えてしまう問題

EC-CUBEの標準機能では、すべての会員（および非会員）に同じ商品一覧が表示されます。

BtoBサイトでは「法人会員にのみ表示する商品カテゴリ」を作りたいケースがあります。たとえば「業務用の大容量品は法人会員のみに表示したい」「プロ向けの専門道具は一般消費者には見せたくない」といった要望です。

また、BtoB専用サイトとして構築する場合、ログインしていない一般ユーザーには価格すら見せたくないというニーズもあります。これも標準機能では対応できません。

### 壁4: 支払い方法を会員種別で変えられない問題

EC-CUBEの標準機能では、支払い方法の設定は「この支払い方法を有効にするかどうか」「最低注文金額・最高注文金額」のみです。「この会員グループには銀行振込のみ表示する」「この会員グループにはクレジットカードを表示しない」といった会員種別ごとの制御ができません。

BtoBサイトでは、クレジットカード決済を利用しない法人がほとんどです。法人間取引では「請求書払い（後払い）」や「銀行振込（前払い）」が主流です。一般消費者向けにはクレジットカードも使えるようにしつつ、法人会員にはそれらを非表示にして銀行振込と後払いのみを表示する、という制御が標準機能ではできません。

### 壁5: 配送方法を会員種別で変えられない問題

支払い方法と同様に、配送方法も会員種別で制御できません。

法人取引では「一括納品（大量の商品を1回で納品する）」「毎週決まった曜日に定期配送する」「倉庫への直送便」など、BtoC向けの一般宅配とは異なる配送オプションが必要になることがあります。「法人会員には業務用一括納品便を選べるようにしたい」「一般消費者に業務用便の選択肢を見せたくない」といった制御が標準機能ではできません。

## 会員グループ管理プラグインとは何か

これら5つの壁をまとめて解決する鍵となるのが「会員グループ管理プラグイン」です。

このプラグインを一言で表すと「会員をグループ分けして、グループごとに異なる体験を提供できる仕組みをEC-CUBEに追加するもの」です。

具体的には、EC-CUBEに「法人会員グループ」「一般会員グループ」「VIP会員グループ」といった複数の会員グループを作成し、会員をいずれかのグループに所属させる機能を追加します。

たとえばこのような設定ができます。

- 「法人会員グループ」に所属している会員には卸価格を表示する
- 「一般会員グループ」に所属している会員には定価を表示する
- 「VIP会員グループ」には特定の限定商品も表示する

プラグイン単体では「会員をグループに分ける」基盤機能を提供します。そこに以下のアドオンを組み合わせることで、BtoB要件を満たす各機能が実現します。

---

## プラグインエコシステムで解決できる全体像

会員グループ管理プラグインとアドオン群を組み合わせた全体構成を整理します。

```
会員グループ管理プラグイン（基盤）
├── 承認制アドオン         → 法人審査フローを実現
├── 価格管理アドオン       → 会員グループ別の卸価格を実現
├── 支払い方法制限アドオン → 会員グループ別の支払い方法制御を実現
└── 配送方法制限アドオン   → 会員グループ別の配送方法制御を実現
```

この構成で、先に挙げた5つの壁が以下のように解決されます。

| BtoBの壁 | 解決するアドオン |
|---|---|
| 会員登録が即時完了してしまう | 承認制アドオン |
| 全員に同じ価格が表示される | 価格管理アドオン |
| 全員に同じ商品が見えてしまう | 価格管理アドオン（表示制御） |
| 支払い方法を会員種別で変えられない | 支払い方法制限アドオン |
| 配送方法を会員種別で変えられない | 配送方法制限アドオン |

コーディングは一切不要で、管理画面の設定だけでこれらの機能を実装できるのがこのプラグインエコシステムの大きな強みです。

次のチャプターからは、会員グループ管理プラグインのインストールと基本設定から順を追って解説していきます。まずは「法人会員グループ」という概念をEC-CUBEに追加することが、すべての出発点になります。
