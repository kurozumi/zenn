## はじめに

EC-CUBEプラグインの開発・販売を行っている筆者ですが、2026年に入ってから売上が大幅に減少しています。

この記事では、なぜEC-CUBEプラグインが売れなくなったのかを分析し、今後のプラグイン開発者としての生存戦略を考察します。

## 売上減少の要因分析

### 1. SaaS型ECプラットフォームへの移行トレンド

最も大きな要因は、**EC-CUBEからShopifyなどのSaaS型プラットフォームへの移行**が加速していることです。

[Web幹事の調査](https://web-kanji.com/posts/eccube-shopify-compare)によると、ECシステムのトレンドは自社でサーバーを持つ「オープンソース型（EC-CUBE）」から、保守管理不要の「SaaS型（Shopify）」へと大きくシフトしています。

**移行が進む理由：**

| 項目 | EC-CUBE | Shopify |
|------|---------|---------|
| サーバー管理 | 自社で必要 | 不要 |
| セキュリティ対応 | 自社で必要 | 自動 |
| アップデート | 手動 | 自動 |
| 初期費用 | 高め | 低め |
| 運用コスト | 変動 | 定額 |

EC-CUBEの年間流通総額は2,100億円（2023年）である一方、makeshopは3,428億円（2025年4月）と、SaaS型プラットフォームが市場シェアを拡大しています。

### 2. AIコーディングツールの普及

[Claude Code](https://claude.ai/claude-code)や[Cursor](https://cursor.sh/)などのAIコーディングツールの普及により、**「プラグインを買う」から「自分で作る」への転換**が起きています。

[Hexabaseの実践レポート](https://www.hexabase.com/column/ai-driven-development-cursor-claude-code-report)では、AIツールを使えばプログラミング初心者でも本格的なWebサイトやECサイトが作れるようになったと報告されています。

これまでプラグインを購入していた層が、以下の理由で自作を選ぶようになっています：

- **コスト削減**：プラグイン購入費 vs AI利用料
- **カスタマイズ性**：自分の要件にぴったり合ったものが作れる
- **学習効果**：AIに教えてもらいながら学べる
- **継続的な改善**：自分で修正・機能追加が可能

```php
// 例：AIに「会員をグループ分けする機能を作って」と依頼するだけで
// 以下のようなコードが生成される時代

namespace Customize\Entity;

use Doctrine\ORM\Mapping as ORM;
use Eccube\Entity\Customer;

/**
 * @ORM\Entity
 * @ORM\Table(name="plg_customer_group")
 */
class CustomerGroup
{
    // ... AIが自動生成
}
```

### 3. 景気・消費動向の影響

[日本経済研究センター](https://www.jcer.or.jp/j-column/column-komine/20260128.html)によると、2026年の実質GDP成長率は前年比+0.8%と緩やかな成長が見込まれています。

しかし、実態としては：

- 西松屋チェーンやカワチ薬品が下方修正
- しまむらは12月の既存店売上高が約2.2%減（9ヶ月ぶりの前年割れ）
- ワークマンも同2.0%減

**大企業・輸出企業・富裕層は上向き、中小企業・低中所得層は下向き**という二極化が深刻化しており、EC-CUBEを利用する中小EC事業者の投資意欲が減退していると考えられます。

### 4. EC-CUBE自体の市場変化

EC-CUBEは「月商1000万円以上で利用されているカートシステム」の利用数でNo.1を獲得していますが、新規導入は減少傾向にあります。

[BiNDec FEED](https://bindec.jp/media/555953136009/)の調査では、BASEはショップ開設数240万店舗以上を誇り、ネットショップ開設実績5年連続No.1を獲得。手軽に始められるプラットフォームに新規参入者が流れています。

## プラグイン別の影響度

売れなくなったプラグインには傾向があります：

### 影響大（売上大幅減）
- **汎用的な機能拡張プラグイン**：AIで代替可能
- **シンプルな管理機能追加**：カスタマイズで対応可能
- **会員管理系**：Shopifyアプリで代替可能

### 影響小（比較的安定）
- **決済連携プラグイン**：専門知識・契約が必要
- **外部サービス連携**：API仕様の理解が必要
- **複雑なビジネスロジック**：AIでも完璧な実装は困難

## 今後の生存戦略

### 1. 高付加価値プラグインへのシフト

AIで簡単に作れない、**専門性の高いプラグイン**に注力する：

- 決済代行サービス連携
- 物流システム連携
- 基幹システム（ERP）連携
- 複雑な在庫管理機能

### 2. サポート・保守サービスの強化

プラグイン単体ではなく、**継続的なサポートをセット**にする：

- インストール・設定サポート
- カスタマイズ対応
- バージョンアップ対応
- トラブルシューティング

### 3. Shopifyアプリへの展開

EC-CUBE市場が縮小するなら、**成長市場へ移動**する選択肢も：

- Shopifyアプリ開発
- その他SaaS型ECプラットフォーム向け開発

### 4. AIを活用した開発効率化

皮肉にも、**AIを使って開発コストを下げる**ことで、価格競争力を維持：

- プラグイン開発の効率化
- ドキュメント作成の自動化
- カスタマイズ対応の迅速化

## まとめ

EC-CUBEプラグインの売上減少は、複数の要因が重なった結果です：

1. **SaaS型ECへの移行加速** - プラットフォーム自体のシェア低下
2. **AIツールの普及** - 「買う」から「作る」への転換
3. **景気の二極化** - 中小EC事業者の投資抑制
4. **新規参入者の減少** - 手軽なプラットフォームへの流出

プラグイン開発者として生き残るためには、AIで代替できない価値を提供するか、成長市場へシフトするか、戦略的な判断が求められています。
