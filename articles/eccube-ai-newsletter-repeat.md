## 「全員に同じメルマガ」、まだ続けていますか？

「配信リストに一斉送信 → 購入ゼロ → また来月…」。

この繰り返しに心当たりがある方は、原因は文章でも商品でもなく**「誰に送るか」の精度**かもしれません。

3 ヶ月購入のない会員に「ありがとうございます」メールを送っても響きません。先週買った会員にセール案内を送れば迷惑です。AI を使えば**誰に・何を・いつ**を個人レベルで自動判定できます。

「でも AI に顧客データを渡すのは怖い」——その心配は正しい感覚です。この記事では**個人情報を一切 AI に渡さず**、EC-CUBE 標準データだけでセグメント配信を実現する方法を解説します。

## 個人情報は渡さなくていい

AI がセグメント分析に必要なのは行動データだけです。

```
× 渡してはいけないデータ
　名前・メールアドレス・住所・電話番号・生年月日

✅ 渡してよいデータ（匿名化済み）
　顧客ID・購入回数・累計購入金額・最終購入日からの経過日数
```

### Claude API のデータポリシー

Anthropic の商用利用規約（Commercial Terms）には以下が明記されています：

> Anthropic may not train models on Customer Content from Services

API 経由で送信したデータはモデルの学習に使用されません。ブラウザの Claude.ai とは別扱いです。

### 個人情報保護法の観点

| 加工方法 | 定義 | 第三者提供 |
|---|---|---|
| **仮名加工情報** | 他の情報と照合しない限り個人を識別できない（顧客 ID への置き換え等） | 原則禁止 |
| **匿名加工情報** | 復元不可能に加工した情報 | 公表義務を果たせば提供可 |

自社サーバーで処理して結果だけ受け取る設計にすれば、仮名加工情報のまま扱えます。ただし顧客 ID + 行動データを外部 API に送信することになるため、自社のプライバシーポリシーに「購買行動の分析に外部 AI サービスを利用する旨」を明記することを推奨します。

## 全体構成

```
EC-CUBE DB（顧客・受注データ）
         │
         ▼
  匿名化処理（自社サーバー内）
  ※ 名前・メール・住所は除外
         │
         ▼
  Claude API（セグメント分析）
  ※ 顧客ID＋行動データのみ
         │
         ▼
  セグメント判定結果を受け取る
         │
         ▼
  メルマガ管理プラグインの送信処理
  ※ オプトアウト管理・履歴記録・バッチ送信を担当
```

名前・メールアドレス・住所は EC-CUBE DB だけにあり、AI には渡りません。

## EC-CUBE 標準データで RFM 分析できる

`Customer` エンティティには RFM 分析に必要なフィールドが**標準で**揃っています。

| フィールド | 内容 | RFM |
|---|---|---|
| `last_buy_date` | 最終購入日 | **R**ecency（最近性） |
| `buy_times` | 購入回数 | **F**requency（頻度） |
| `buy_total` | 累計購入金額 | **M**onetary（金額） |
| `create_date` | 会員登録日 | 在籍期間 |
| `point` | 保有ポイント | エンゲージメント |

これらは受注完了のたびに自動更新されます。追加の集計処理は不要です。

## AI が判定するセグメントの例

匿名化した購買データを Claude API に渡すと、以下のようなセグメントに分類できます。

| セグメント | 条件の例 |
|---|---|
| **VIP** | 購入回数 5 回以上かつ累計購入金額 5 万円以上 |
| **アクティブ** | 最終購入から 60 日以内 |
| **離脱予備軍** | 最終購入から 60〜90 日 |
| **新規** | 購入回数 1 回 |
| **離脱** | 最終購入から 90 日超 |

セグメントごとに件名・本文・クーポンを変えることで、同じ商品でも受け取る側の状況に合ったメッセージを届けられます。

## 公式メルマガ管理プラグインで送信基盤を揃える

EC-CUBE 公式の**メルマガ管理プラグイン**（[GitHub](https://github.com/EC-CUBE/mail-magazine-plugin)）を使えば、以下を自前実装せずに済みます。

- **配信停止（オプトアウト）管理** — 会員登録時・マイページで受信可否を設定
- **テンプレート管理** — セグメントごとのテンプレートを管理画面から編集
- **送信履歴の記録** — 誰にいつ何を送ったかを管理画面で確認
- **バッチ送信** — 大量配信を 100 件ずつ処理
- **失敗時の再送** — 送信失敗した会員へのリトライ

:::message alert
特定電子メール法により、受信拒否の意思を示した宛先への送信は違法です。プラグインが提供するオプトアウト管理機能を必ず活用してください。
:::

## AI とプラグインの役割分担

| 役割 | 担当 |
|---|---|
| 誰に送るか（セグメント判定） | Claude API |
| 配信停止管理 | メルマガ管理プラグイン |
| テンプレート管理 | メルマガ管理プラグイン（管理画面で編集） |
| 送信履歴の記録 | メルマガ管理プラグイン |
| バッチ送信・再送 | メルマガ管理プラグイン |
| 定期実行 | Symfony Console コマンド + cron |

AI は「誰に送るか」だけに集中し、送信インフラはプラグインに任せるシンプルな設計です。

## まとめ

EC-CUBE には RFM 分析に必要な購買データが標準で揃っており、AI に渡すのは匿名化した行動データだけで十分です。公式メルマガ管理プラグインと組み合わせることで、個人情報を外部に出さずにセグメント別の自動配信が実現できます。

「全員に同じメルマガ」から「その人に届くメルマガ」へ。EC-CUBE とAI を組み合わせれば、大掛かりなマーケティングツールを導入しなくても実現できます。
