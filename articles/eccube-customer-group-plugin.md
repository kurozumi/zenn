## この記事でわかること（TL;DR）

- EC-CUBE 4.3 標準にない「会員グループ」をプラグインで追加する方法
- 商品・カテゴリ・ページ・カートの全経路でグループ別閲覧制限をかける設定手順
- BtoB 取引先限定・会員ランク制・法人専用商品など 5 つのユースケース
- Voter + Gate パターンで実現するアクセス制御の仕組み

## こんな課題はありませんか？

- 取引先 A 社には A 社向け商品だけを見せたい。でも同じ EC-CUBE で管理したい
- プラチナ会員にだけ限定商品ページを表示したい
- 法人審査済みアカウント以外からのカート追加を、URL を直打ちされても防ぎたい

EC-CUBE 4.3 はこれらを標準機能では実現できません。**しかし、`CustomerGroup` プラグイン 1 本をインストールするだけで、商品・カテゴリ・ページ・カート追加のすべての経路でグループ別のアクセス制御が動き出します。**

この記事では、`ec-cube/customergroup42` の設定手順と内部アーキテクチャを解説します。

## プラグインの概要

- **対応バージョン**: EC-CUBE 4.2 / 4.3、PHP 7.4 〜 8.3
- **ライセンス**: LGPL-2.1
- **リポジトリ**: https://github.com/kurozumi/eccube-plugin-customer-group

## 主な機能一覧

| 機能 | 概要 |
|------|------|
| 会員グループ管理 | グループの作成・編集・削除・並び替え |
| 商品の閲覧制限 | グループを指定した商品を非所属会員・非会員から非表示 |
| カテゴリの閲覧制限 | 親カテゴリ単位でグループを設定して閲覧制限 |
| ページの閲覧制限 | ユーザー作成ページ（`EDIT_TYPE_USER`）に閲覧制限 |
| カートへの追加制限 | 非所属会員がカートに商品を追加しようとするとエラー |
| 非会員表示オプション | グループ単位で非会員への表示を許可するフラグ |
| Option1 モード | 会員は所属グループの商品のみ表示（それ以外は非表示） |
| CSV インポート | 会員グループ・商品グループの一括登録 |
| 管理画面絞り込み | 会員・商品・受注をグループで絞り込み検索 |

## データベース設計

プラグインは以下のテーブルを追加します。

### グループマスタ: `plg_customer_group`

| カラム | 説明 |
|--------|------|
| `id` | グループID（PK） |
| `name` | グループ名（フロント表示用） |
| `backendName` | バックエンド表示名（管理画面用） |
| `optionNonMemberDisplay` | 非会員にも表示するかどうかのフラグ |
| `sortNo` | 並び順 |

### プラグイン設定: `plg_customer_group_config`

| カラム | 説明 |
|--------|------|
| `optionGroupProductHidden` | 商品の閲覧制限を有効にするか（デフォルト: `true`） |
| `optionGroupCategoryHidden` | カテゴリの閲覧制限を有効にするか（デフォルト: `false`） |
| `option1` | Option1 モードを有効にするか（デフォルト: `false`） |

### 中間テーブル（多対多）

- `plg_customer_group_customers_groups`: 会員 ↔ グループ
- `plg_customer_group_products_groups`: 商品 ↔ グループ
- `plg_customer_group_categories_groups`: カテゴリ ↔ グループ
- `plg_customer_group_pages_groups`: ページ ↔ グループ
- `plg_customer_group_orders_groups`: 受注 ↔ グループ

## アーキテクチャの特徴

### EntityExtension（トレイト注入）

このプラグインは EC-CUBE の `EntityExtension` 機能を活用して、既存エンティティにグループ関連メソッドを追加します。

```php
// Eccube\Entity\Customer に注入されるトレイト
trait CustomerTrait
{
    public function getGroups(): Collection { ... }
    public function getGroupProducts(): Collection { ... }
    public function getGroupCategories(): Collection { ... }
    public function getGroupPages(): Collection { ... }
    public function hasGroups(): bool { ... }
}
```

EC-CUBE 本体の `Customer` エンティティを改変することなく、`Customer::getGroups()` や `Customer::hasGroups()` が利用できるようになります。

### Voter + Gate パターンによるアクセス制御

アクセス制御には Symfony の **Voter** と独自の **Gate（Chain of Responsibility）** パターンを組み合わせています。

```
ProductVoter
  └─ GateContext
       ├─ NonMemberGate    (priority: 100) : 非会員向け判定
       ├─ ProductGate      (priority: -100): 会員の商品グループ判定
       └─ Option1Gate      (priority: -101): Option1モード判定
```

`GateContext` に登録された Gate を優先度順に評価し、一つでも `false` を返すとアクセス拒否（`Symfony\Component\Security\Core\Exception\AccessDeniedException`）になります。

### QueryCustomizer による商品一覧の絞り込み

フロントの商品一覧（EC-CUBE 本体の `Eccube\Repository\QueryKey::PRODUCT_SEARCH`）をカスタマイズして、ログイン状態とグループ所属に応じた SQL サブクエリを動的に追加します。

- **グループ会員（通常モード）**: 他グループに紐づく商品を除外するサブクエリを追加
- **グループ会員（Option1 モード）**: 自分のグループに紐づく商品のみを表示するサブクエリを追加
- **非会員**: グループに紐づいているすべての商品を除外するサブクエリを追加

### PurchaseFlow Validator

カート投入時にも `GroupValidator` が動作し、閲覧制限されている商品をカートに入れようとすると数量を 0 にリセットしてエラーメッセージを表示します。

## 設定・使い方

### 1. プラグインのインストール

EC-CUBE の管理画面（または Composer）からプラグインをインストールし、有効化します。

### 2. 基本設定

管理画面 → **会員グループ設定** → **基本設定** から以下を設定します。

| 設定項目 | 説明 |
|----------|------|
| 商品の閲覧制限 | グループを設定した商品を非所属会員・非会員から非表示にする |
| カテゴリの閲覧制限 | グループを設定したカテゴリを非所属会員・非会員から非表示にする |
| Option1 モード | 会員を所属グループの商品のみ表示するモードに切り替える |

### 3. 会員グループの作成

管理画面 → **会員管理** → **会員グループ一覧** → **会員グループ登録** からグループを作成します。

- **グループ名**: フロント表示用の名称
- **バックエンド表示名**: 管理画面での識別用名称
- **非会員表示オプション**: グループ設定された商品を非会員にも見せる場合に有効化

### 4. 会員へのグループ割り当て

管理画面 → **会員管理** → 対象会員の編集画面を開き、「会員グループ」チェックボックスから所属グループを選択します。

CSVを使った一括登録にも対応しています。

### 5. 商品へのグループ設定

管理画面 → **商品管理** → 対象商品の編集画面を開き、「閲覧可能な会員グループ」チェックボックスからグループを選択します。

グループを設定すると、そのグループに所属していない会員・非会員からはその商品が非表示になります（基本設定で商品の閲覧制限が有効の場合）。

### 6. カテゴリへのグループ設定

管理画面 → **商品管理** → **カテゴリ管理** から対象カテゴリ（**親カテゴリのみ**）の編集画面を開き、「閲覧可能な会員グループ」を設定します。

:::message
カテゴリの閲覧制限は**親カテゴリのみ**設定可能です。子カテゴリへの個別設定はサポートされていません。
:::

## 具体的なユースケース

### ユースケース 1: BtoB サイトの取引先限定商品

複数の取引先企業（A社、B社、C社）に対してそれぞれ専用商品ラインナップを提供するケースです。

**設定例:**
1. グループ「A社」「B社」「C社」を作成
2. 各取引先の担当者アカウントをそれぞれのグループに割り当て
3. A社向け商品に「A社」グループを設定

**結果:** A社の担当者がログインすると A社向け商品のみ表示され、B社・C社の商品は見えません。

---

### ユースケース 2: 会員ランクに応じたプレミアム商品

「一般会員」「ゴールド会員」「プラチナ会員」といったランク制を設けて、上位会員にのみ限定商品・限定価格を表示するケースです。

**設定例:**
1. グループ「ゴールド会員」「プラチナ会員」を作成
2. 購入金額や回数に応じて会員を手動またはCSVで各グループへ割り当て
3. プレミアム商品に対象グループを設定

**結果:** ゴールド会員以上にのみ表示される限定商品ページを実現できます。

---

### ユースケース 3: 法人向けサイトで個人購入を防止

「法人会員」グループのみが購入できる商品を設定し、個人アカウントや非会員からの購入を制限するケースです。

**設定例:**
1. グループ「法人会員」を作成
2. 法人審査済みのアカウントを「法人会員」グループに割り当て
3. 法人向け商品に「法人会員」グループを設定

**結果:** 一般会員が該当商品ページを開いても表示されず、カートへの追加も `GroupValidator` がブロックします。

---

### ユースケース 4: 会員限定コンテンツページ

特定の会員グループにのみ閲覧を許可するコンテンツページ（使い方ガイド・限定動画ページなど）を作成するケースです。

**設定例:**
1. グループ「プレミアム会員」を作成
2. ページ管理でユーザー作成ページ（`EDIT_TYPE_USER`）を作成
3. そのページに「プレミアム会員」グループを設定

**結果:** 非会員や一般会員がそのページにアクセスしようとするとアクセス拒否になります。

---

### ユースケース 5: Option1 モードによる完全な商品ラインナップ分離

Option1 モードを有効にすると、会員は**自分のグループに紐づく商品だけ**が表示されます。グループ設定のない商品も含め、無関係な商品が一切見えなくなります。

:::message alert
Option1 モードを有効にすると、グループに割り当てられていない会員には**すべての商品が非表示**になります。既存の会員がグループ未割り当ての状態で本番環境に適用すると、その会員から全商品が見えなくなるため、事前に全会員のグループ割り当てを完了してから有効にしてください。
:::

**向いているケース:**
- 特約店向けポータルサイト（A代理店・B代理店それぞれ専用の商品ページ）
- フランチャイズ本部が各店舗向けに発注サイトを運営する場合

## まとめ

CustomerGroup プラグインは以下の課題を解決します。

- ✅ BtoB サイトで取引先ごとに商品ラインナップを分けたい
- ✅ 会員ランクに応じてプレミアム商品・限定コンテンツを表示したい
- ✅ 法人会員専用商品を一般会員・非会員から完全に隠したい
- ✅ 会員限定のコンテンツページを作りたい

Symfony の Voter パターンと EC-CUBE の QueryCustomizer を組み合わせた実装で、商品一覧・商品詳細・カテゴリ・ページ・カート追加の各経路でアクセス制御が機能します。

:::message
閲覧制限はあくまでアプリケーション層での制御です。`web/` 配下に直接配置された画像・ファイルへのURLアクセスや、EC-CUBE の REST API 経由のアクセスには別途対策が必要な場合があります。機密性の高い情報は、アプリケーション層の制御だけに頼らず、サーバー設定や認証の多層防御を組み合わせてください。
:::
