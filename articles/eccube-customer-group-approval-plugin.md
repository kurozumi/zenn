EC-CUBE の標準機能では、会員登録フォームを送信すると誰でも即座に会員になれます。BtoB サイトや法人向けショップでは、これが問題になることがあります。

- **取引先ではない企業が登録してしまった**
- **信頼できる取引先だけに卸価格を見せたい**
- **会員登録前に担当者が審査を行いたい**

このような課題を解決するのが、**会員グループ管理【BtoB】::会員登録承認制アドオン**です。会員登録後に管理者が承認するまで本会員にならない「承認制」の登録フローを EC-CUBE に追加できます。

## 承認制にするとどうなるか

承認制を有効にすると、会員登録フローが以下のように変わります。

**通常の登録フロー（EC-CUBE 標準）:**
```
会員登録 → メールアドレス確認 → 本会員
```

**承認制フロー（このアドオン導入後）:**
```
会員登録 → 承認待ち画面
         ├─ 管理者に承認依頼メール送信
         └─ 会員に仮登録通知メール送信
                    ↓
             管理者が承認URLをクリック
                    ↓
          本会員化 + 会員に完了メール送信
```

会員が登録フォームを送信すると、管理者に**承認依頼メール**、会員に**仮登録通知メール**が同時に送信されます。管理者がメール内の承認 URL をクリックするだけで承認が完了し、会員に**本会員登録完了メール**が送信されます。承認するまで会員はログインできないため、審査前の取引先が商品情報や卸価格を閲覧することを防げます。

## 解決できる課題

### BtoB サイトの会員審査

卸売・取引先向けのショップで、登録申請のあった企業を担当者が審査してから会員にしたいケースです。登録フォームに会社名・担当者名・電話番号などを記入してもらい、内容を確認してから承認することで、信頼できる取引先のみに卸価格や限定商品を公開できます。

### 法人会員限定の特典管理

法人会員のみが使える割引価格やカテゴリへのアクセスを承認制で管理するケースです。[会員グループ管理プラグイン](https://zenn.dev/kurozumi/articles/eccube-customer-group-plugin)と組み合わせることで、承認済みの法人会員だけが特定の商品・カテゴリ・価格を閲覧できる環境を構築できます。さらに[会員グループ価格管理アドオン](https://zenn.dev/kurozumi/articles/eccube-customer-group-price-plugin)を追加すると、法人会員向けの掛け率・割引率も設定できます。

### 招待制・クローズドな EC サイト

一般公開はしているが、実際に購入できるのは承認済みの会員のみ、というクローズドな運用に適しています。

## 必要なプラグイン（3つ）

このアドオンは単体では動作しません。以下の順番でインストールしてください。

| 順番 | プラグイン | 役割 |
|------|-----------|------|
| 1 | 会員グループ管理プラグイン | グループ管理の土台 |
| 2 | 会員グループ管理【BtoB】::会員登録アドオン | 登録時にグループへ自動割り当て |
| 3 | 会員グループ管理【BtoB】::会員登録承認制アドオン | 承認制フローの追加 |

:::message
インストール順を間違えると有効化できません。必ず上から順に有効化してください。
:::

## 導入手順

### Step 1: 会員グループ管理プラグインをインストール

まだインストールしていない場合は、EC-CUBE オーナーズストアから[会員グループ管理プラグイン](https://www.ec-cube.net/products/detail.php?product_id=2439)をインストールして有効化します。

### Step 2: 会員登録フォームアドオンをインストール

[会員グループ管理【BtoB】::会員登録アドオン](https://www.ec-cube.net/products/detail.php?product_id=2442)をインストールして有効化します。

このアドオンを有効化すると、グループの設定画面に「**会員登録時に会員グループ登録**」というトグルが追加されます。

### Step 3: 承認制アドオンをインストール

[会員グループ管理【BtoB】::会員登録承認制アドオン](https://www.ec-cube.net/products/detail.php?product_id=2443)をインストールして有効化します。

有効化すると以下が自動で作成されます。
- 承認待ち画面（`/entry/approval`）
- 管理者宛て承認依頼メールテンプレート
- 会員宛て仮登録通知メールテンプレート

### Step 4: グループの設定

管理画面 → **会員管理** → **会員グループ一覧** → 対象グループの編集画面を開き、以下の2つを有効にします。

| 設定項目 | 説明 |
|----------|------|
| 会員登録時に会員グループ登録 | 登録フォーム送信時にこのグループへ自動割り当て |
| 会員登録を承認制にする | 承認なしに本会員にならないフローを有効化 |

:::message
「会員登録時に会員グループ登録」は**1グループのみ**設定できます。複数グループへの同時設定はサポートされていません。
:::

### Step 5: 動作確認

設定後、実際に会員登録フォームからテスト登録を行い、以下を確認してください。

1. 登録完了後に承認待ち画面（`/entry/approval`）が表示される
2. 管理者メールアドレスに承認依頼メールが届く
3. 登録者にも仮登録通知メールが届く
4. メール内の承認 URL をクリックすると本会員化される
5. 会員に本会員登録完了メールが届く

:::message
テスト登録後は管理画面の会員一覧から仮会員データを必ず削除してください。本番環境に不要な登録データを放置すると個人情報管理上の問題になります。
:::

## 承認の方法

承認は**メール内の URL をクリックするだけ**です。管理画面に専用の承認ページはありません。

1. 管理者宛てメールを開く
2. メール内の承認 URL（アクティベーション URL）をクリック
3. 自動的に会員が本会員化され、会員に完了メールが送信される

承認を保留したい場合はメールを放置するだけでよく、会員は仮会員のまま（ログイン不可）の状態が続きます。承認待ちの会員は管理画面の会員一覧で「仮会員」ステータスとして確認できます。不審な申請は管理画面から手動で削除することを推奨します。

:::message alert
承認 URL はメール本文にそのまま記載されます。管理者メールアカウントへの不正アクセスがあると、第三者が承認 URL を悪用する可能性があります。管理者メールアカウントには二段階認証を設定してください。
:::

## まとめ

会員グループ管理【BtoB】::会員登録承認制アドオンを導入することで、以下が実現できます。

- ✅ BtoB サイトで取引先の審査が完了するまで会員登録を保留できる
- ✅ 管理者がメール1本で承認できる簡単な運用フロー
- ✅ 承認済み会員のみが卸価格・限定商品にアクセスできる環境を構築できる
- ✅ [会員グループ管理プラグイン](https://zenn.dev/kurozumi/articles/eccube-customer-group-plugin)の閲覧制限・[価格管理アドオン](https://zenn.dev/kurozumi/articles/eccube-customer-group-price-plugin)と組み合わせて本格的な BtoB EC サイトを構築できる
