## GA4 Data API で取得できるデータ


GA4 Data API のファネルレポートエンドポイント（`runFunnelReport`）を使うと、ステップごとの以下のデータを取得できます。

- **アクティブユーザー数**: 各ステップに到達したユーザー数
- **完了率（Completion Rate）**: 前のステップから次のステップへ進んだ割合
- **離脱数（Abandonments）**: 各ステップで離脱したユーザー数
- **離脱率（Abandonment Rate）**: 各ステップでの離脱割合

さらにディメンションで分解することで、以下のような切り口で分析できます。

- **デバイス別**: スマホとPCで離脱率に差がある
- **流入チャネル別**: 広告経由のユーザーはカートに入れやすいが完了しにくい
- **時間帯別**: 深夜のユーザーは支払ステップで離脱しやすい

ℹ️ `runFunnelReport` はアルファ版（`v1alpha`）のエンドポイントです。本番環境での利用は仕様変更の可能性を考慮してください。

## GA4 のデータに個人情報は含まれるか

**含まれません。** GA4 Data API のレスポンスは集計データ（ステップ別のユーザー数・割合）であり、個人を特定できる情報は含まれない設計です。

Google のポリシーとして、メールアドレスや電話番号などの個人情報（PII）を GA4 に送信することは禁止されています。Data API で取得できるのはその集計結果のみです。

## AI が分析できること

GA4 から取得したファネルデータを Claude API に渡すと、以下のような分析が可能です。

### 1. 離脱パターンから原因を推定する

カート追加: 1,000人
チェックアウト開始: 600人（離脱率 40%）
住所・支払入力: 200人（離脱率 67%）← ここが最大の離脱ポイント
注文完了: 150人（離脱率 25%）

このデータを AI に渡すと：

> 住所・支払方法入力ステップでの離脱率が 67% と突出して高い。一般的な原因として以下が考えられます：
> 1. 入力項目が多くフォームが長い
> 2. ゲスト購入ができず会員登録が必須
> 3. 希望する支払方法（コンビニ払い・後払い等）に対応していない
> 4. スマホでの入力が煩雑（オートフィル非対応等）

という形で、数字のパターンから考えられる原因を列挙してくれます。

### 2. デバイス別の差異から施策を絞る

スマホ: 住所入力の離脱率 78%
PC: 住所入力の離脱率 45%

AI に渡すと：

> スマホとPCの離脱率に 33 ポイントの差があります。スマホ特有の入力課題（キーボード切り替え・フォームの見切れ等）が主因の可能性が高く、EC-CUBE の入力フォームのモバイル最適化を優先すべきです。

### 3. 改善施策を優先度付きで提案させる

ファネル全体のデータと「ECサイトの業種・客単価・ターゲット層」を加えて渡すと、施策の優先度まで整理してくれます。

> **優先度 高**: ゲスト購入の導入（住所入力前のログイン必須を解除）
> **優先度 中**: 支払方法の追加（コンビニ払い・バンドルカード等）
> **優先度 低**: フォームの入力補助強化（郵便番号による住所自動入力等）

Symfony Console コマンドとして実装すれば、定期的に GA4 データを取得して AI 分析レポートをメール送信する、という自動化も可能です。

## メルマガ施策との連携

前回の記事で紹介した AI メルマガセグメント配信と組み合わせると、さらに強力になります。

- **ファネル分析 AI**: 「住所入力で離脱しているスマホユーザーが多い」と特定
- **メルマガ AI**: カート離脱後のフォローメール（「お買い物を続ける」リンク）をセグメント送信

GA4 の離脱データで「どこで・誰が・なぜ」を把握し、メルマガで「その人に合ったフォロー」を送る——この 2 つを AI で自動化することで、CVR 改善のサイクルが回り始めます。

## まとめ

GA4 Data API は集計データを返すため、個人情報を AI に渡すことなくファネル分析が可能です。

EC-CUBE の購入フロー（カート → チェックアウト → 住所・支払入力 → 完了）のステップ別データを Claude API に渡すことで、**人間が時間をかけて行っていた「データを読んで仮説を立てる」作業を自動化**できます。

「数字は見ているけど、何をすればいいかわからない」という状態から抜け出すヒントになれば幸いです。
