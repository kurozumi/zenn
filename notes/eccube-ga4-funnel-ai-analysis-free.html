GA4のファネルデータをAIに投げるだけで改善施策が出てくる話

この記事は EC-CUBE 4.3 以上を対象としています。
また、Claude Code を使って書かれています。内容に誤りがある場合はコメントでお知らせください。

GA4 のファネルを見て「で、何をすればいいんだ」となっていませんか

「カート追加後に 70% が離脱している」

この数字を見て、次にとる行動がすぐ浮かびますか？

フォームが長いのか、支払方法が足りないのか、スマホ対応が悪いのか——原因の候補は複数あって、どこから手をつければいいかわからない。結局「要調査」で止まってしまうループ。

GA4 Data API でファネルデータを取得し、Claude API に渡すと「ステップ別の離脱率パターンから考えられる原因と、優先度付きの改善施策」を自動で出力できます。

個人情報は一切 AI に渡しません。GA4 Data API のレスポンスは集計データであり、個人を特定できる情報は含まれない設計になっています。

EC-CUBE の購入フローと GA4 イベントの対応

EC-CUBE の購入フローは以下のステップで構成されています。

ステップ EC-CUBE のルート GA4 イベント
商品詳細 product_detail view_item
カート追加 cart add_to_cart
チェックアウト開始 cart_buystep begin_checkout
ログイン shopping_login （計測対象外）
住所・支払方法入力 shopping add_shipping_info / add_payment_info
注文確認 shopping_confirm （計測対象外）
注文完了 shopping_complete purchase

GA4 のeコマース計測で使う標準イベントの順序はこうなります。

add_to_cart
    ↓
begin_checkout
    ↓
add_shipping_info
    ↓
add_payment_info
    ↓
purchase

各ステップのユーザー数と離脱率を GA4 のファネル探索レポートで可視化できます。
